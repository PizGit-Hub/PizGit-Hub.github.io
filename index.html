📊 1. Data Representation / 数据表示
二进制前缀 (Binary Prefixes)：十进制前缀 1 KB = 1000 B, 1 MB = 10⁶ B (硬盘常用)。二进制前缀 1 KiB = 1024 B, 1 MiB = 1024² B (RAM精确)。重要性：RAM使用二进制前缀，存储设备常使用十进制前缀以免混淆。
Binary Prefixes: Denary prefixes: 1 KB = 1000 B, 1 MB = 10⁶ B (common for storage). Binary prefixes: 1 KiB = 1024 B, 1 MiB = 1024² B (used for RAM). Important: RAM uses binary prefixes, hard drives often use denary prefixes.
数制 (Number Systems)：二进制(base-2)，十进制(base-10)，十六进制(base-16)。转换：二进制↔十进制(位权)，二进制↔十六进制(四位分组)，十六进制↔十进制(乘16)。
Number Systems: Binary (base-2), Denary (base-10), Hexadecimal (base-16). Conversions: binary↔denary (place value), binary↔hex (nibble grouping), hex↔denary (×16).
二进制编码十进制(BCD)：4位表示一位十进制数，如2500 → 0010 0101 0000 0000。常用于计算器、数字钟、财务系统(避免舍入误差)。
Binary Coded Decimal (BCD): 4 bits per denary digit, e.g. 2500 → 0010 0101 0000 0000. Used in calculators, digital clocks, financial systems (avoids rounding errors).
一补码与二补码：一补码按位取反(存在+0/-0)；二补码取反加一，单零，8位范围-128~+127，统一加法器，硬件简单。
One's & Two's Complement: One's complement inverts bits (has +0/-0). Two's complement inverts then adds 1 (single zero, range -128 to +127 for 8-bit), simpler hardware, same adder circuit.
二进制加法与溢出：逢二进一，超出位宽时发生溢出，可能改变符号位。
Binary Addition & Overflow: Carry when sum ≥2; overflow occurs when result exceeds bit width, may flip sign bit.
十六进制应用：MAC地址、颜色代码(#66FF33)、URL编码(%xx)等。
Hexadecimal Uses: MAC addresses, colour codes (#66FF33), URL encoding (%xx).
字符集：ASCII(7-bit,128字符)，扩展ASCII(8-bit,256字符)，Unicode(≥16-bit)支持全球语言与表情符号。
Character Sets: ASCII (7-bit,128 chars), Extended ASCII (8-bit), Unicode (min 16-bit) supports all languages & emoji.
🎨 2. Multimedia / 多媒体
位图图像 (Bitmap)：由像素组成，图像分辨率=宽×高，颜色深度(每像素位数)决定颜色数=2ⁿ。文件大小(位)=分辨率×颜色深度。
Bitmap Images: Composed of pixels; image resolution = width × height; colour depth (bits per pixel) → number of colours = 2ⁿ. File size (bits) = resolution × colour depth.
矢量图形 (Vector)：由数学方程定义，存储绘图指令，无限缩放无失真，适合Logo、图标、字体。常见格式SVG, AI, EPS。
Vector Graphics: Defined by mathematical equations (drawing list), infinitely scalable with no loss, ideal for logos, icons, fonts. Common formats: SVG, AI, EPS.

声音编码 (Sound Encoding)：模拟信号通过采样(采样率Hz，采样分辨率bit)转为数字。CD质量:44.1kHz/16bit。质量越高文件越大。
Sound Encoding: Analogue sound is sampled (sampling rate in Hz, sampling resolution in bits). CD quality: 44.1 kHz / 16-bit. Higher quality → larger file size.
🗜️ 3. Compression / 压缩
有损压缩 (Lossy)：数据永久丢失，不可逆，高压缩率(JPEG, MP3)。适合图像、音频、视频(可接受质量损失)。
Lossy Compression: Data lost permanently, irreversible, high compression ratio (JPEG, MP3). Suitable for images, audio, video (acceptable quality loss).
无损压缩 (Lossless)：完全可逆，压缩率较低，文档、程序必需(RLE, DOCX, PDF)。
Lossless Compression: Fully reversible, lower compression ratio, required for documents, programs (RLE, DOCX, PDF).
游程编码 (RLE)：将连续重复元素替换为计数+值，如"AAAABBCCDAA" → 4A3B2C1D2A，大幅节省存储空间。
Run-Length Encoding (RLE): Replaces consecutive identical elements with count+value, e.g., "AAAABBCCDAA" → 4A3B2C1D2A, saves significant storage.
常见压缩格式：MP3(音频有损)，MP4(多媒体容器)，JPEG(位图图像有损)。
Common Compressed Formats: MP3 (lossy audio), MP4 (multimedia container), JPEG (lossy bitmap images).
